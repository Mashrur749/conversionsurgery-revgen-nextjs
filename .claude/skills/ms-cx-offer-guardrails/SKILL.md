# CX and Offer Guardrails

Use for all changes that alter client-visible behavior, billing, guarantees, or reporting.

## Guardrails
1. Never ship behavior that contradicts sold promises in the active offer document.
2. Prefer transparent UX copy over ambiguous automation states.
3. Ensure fallback/error states do not create silent failures for clients.
4. Preserve compliant-hours legal qualifiers where required.
5. Avoid manipulative wording in client-facing reports and notifications.

## Offer Parity Check
Before finalizing changes, confirm:
- Promise impacted
- Behavior implemented
- Client-facing copy aligned
- Operations workflow updated
